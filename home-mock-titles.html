Modular personal software

Every Stella surface, working as one system.

Intro for the product grid. One assistant reaches your chat, browser, files, voice, phone, and desktop; each surface stays familiar but the work flows through the same Stella running on your computer.

  1. Conversation

    One chat can run several jobs at once.

    Replies stream in while background work stays visible. Stella keeps moving through independent tasks in the same ongoing chat.

    Mock: animated multitask chat (streaming reply + working indicator + parallel tasks).

  2. Computer use

    Stella can drive your Mac in the background.

    Let Stella move through real app windows while you keep working, from calendars and mail to the files already on your machine.

    Mock: fully-designed desktop + app window + live automation step list.

  3. Browser use

    Research happens inside a full browser context.

    Stella can compare pages, read tables, and carry the result back into the same conversation.

    Mock: full browser window + Stella browser extension console + cursor.

  4. Voice and dictation

    Talk to Stella, or dictate into any app.

    Speak naturally, or drop a compact dictation bar over the message, note, or document you are already writing.

    Mock: dictation overlay over a real app + waveform; voice-agent mode.

  5. Documents

    Word, Excel, PowerPoint, and PDFs are first-class work.

    Stella can produce files, summarize them, and show the result in the workspace panel or chat artifact surface.

    Mock: artifact list with real file-type icons + spreadsheet/doc/deck/PDF preview.

  6. Phone and connectors

    Your phone reaches the same Stella on your computer.

    iMessage, Telegram, Discord, and the Stella app can all route to your paired desktop when it is online.

    Mock: phone surfaces for iMessage / Telegram / Discord / Stella app + relay flow.

Self-modifying interface

Stella can stop looking like Stella.

Five completely different app shapes Stella could build around a person's taste, each heavily inspired by a well-known real-world product.

  1. Apple Music-inspired library

    For playlist-minded planners who group life into mixes.

  2. Xbox-inspired home

    For achievement-driven people who want tasks to feel like quests.

  3. Notion-inspired database

    For notebook people who think in pages, tables, and checklists.

  4. Linear-inspired queue

    For operators who want crisp queues and almost no decoration.

  5. Ableton-inspired timeline

    For makers who see routines as tracks, clips, and levels.

Local runtime

The backbone is the computer you already use.

Stella is not a separate SaaS workspace. The desktop runtime keeps memory, tools, generated files, phone routes, and background agents tied to the same local Stella.

  1. Local

    Files, conversations, and memory stay on the computer.

  2. Open

    Runtime, prompts, skills, and UI are editable.

  3. One

    Every surface routes back to the same ongoing Stella.

  4. Any

    Chat, browser, phone, voice, files, and computer use connect.

Extensible by design

Use Stella as-is, connect more, or change the app itself.

  1. Open and ask

    Zero setup for normal use. Start with the desktop app and keep one ongoing conversation.

  2. Bring your stack

    Use managed access, your own provider keys, local models, or Claude Code where supported.

  3. Install what others build

    Store-backed integrations and shared add-ons can extend Stella without changing the core app by hand.

  4. Change the app itself

    Runtime, prompts, skills, UI, and workflows are editable because Stella is open source.
