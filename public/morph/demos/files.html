Can you walk me through how auth.ts validates a session token?

The flow starts in validateSession: it reads the bearer token, verifies the signature against your signing key, and then checks the expiry claim before anything else runs.

If the token is valid, it loads the matching user record and attaches it to the request. If any step fails, it throws an AuthError so the route handler can return a clean 401.
